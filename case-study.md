# Mode: case-study — Evaluate Case Study Angle

Read `agency.md` and `case-studies.md` before starting.

## Trigger
User wants to evaluate whether a completed project makes a good case study and how to frame it.

## Process

Evaluate the project on:
1. **Outcome clarity** — Is there a measurable outcome to highlight?
2. **Transferability** — Will this resonate with target prospects?
3. **Proof value** — Does it demonstrate a capability we want to sell more of?
4. **Permission** — Can we publish it? (Ask Mordecai about client consent)

Output:
- Case study fit score
- Recommended headline (problem → solution → result format)
- Full case study draft (300–500 words, publishable quality)
- Suggested use cases: proposal insert, website page, LinkedIn post, outreach proof point

Append approved case studies to `case-studies.md`.
